Writing Functions (pt. 1)

Function declaration

  • loaded before any code is executed
  • causes the entire function definition to be 'hoisted'
alert(foo()); // Alerts 5.
function foo() { return 5; }

Writing Functions (pt. 2)

Function expression

  • only loaded when we specifically ask for them
  • only the name gets hoisted, not the initialization
alert(foo()); // ERROR! foo wasn't loaded yet
var foo = function() { return 5; }

Writing Functions (pt. 3)

Function Literal (IIFE)

  • invoked immediately
  • evaluated as an expression
(function() {
  // do stuff immediately
})()

Jargon

null vs. undefined

Null: deliberately set value of nothing; intentional placeholder

Undefined: value of nothing because you have not provided one

var scores = {
  "james": null,
  "sarah": 90
}

console.log(scores.james) // returns null
console.log(scores.bryan) // returns undefined

Jargon

args vs. params

Arguments: used to invoke a function

Parameters: used to define a function

refreshCart(these, are, args);

function refreshCart(these, are, params) {
  // do stuff here
};

Jargon

Method vs. function

Method: a function that is defined as a property of an object. This object is implicitly passed as the context for the function.

function refreshCart() { // this is a function }
var cart = {
  items: ["milk","eggs","cereal"]
  refresh: function() {
    // refresh is a method
    // this.items would equal cart.items
  }
}
