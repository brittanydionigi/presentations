Healthy vs. Habit Hacks

Healthy

  • only when absolutely necessary
  • no better solution at the time; quick fix for live code
  • commit message explaining why it happened/how it should be fixed

Habit

  • little or no explanation for the new code
  • doesn't conform to style standards/doesn't validate
  • no follow-up or future plans for correcting it
